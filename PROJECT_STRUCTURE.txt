ARQUITECTURA
============

Patrón: Clean Architecture
Capas:
  1. Domain (Entidades): Núcleo del negocio, sin dependencias
  2. Application (Usecases): Lógica de negocio, interfaces
  3. Infrastructure (Repositorios): Implementación de acceso a datos
  4. Delivery (Handlers): Presentación HTTP

Flujo de una solicitud:
  HTTP Request → Middleware → Handler → Service → Repository → DB
                    ↑                                           ↓
                    ←───── Response JSON ←────────────────────

CARACTERÍSTICAS PRINCIPALES
===========================

✓ JWT Authentication con access y refresh tokens
✓ Rate Limiting: 100 req/min por IP
✓ Row-Level Security en PostgreSQL (RLS)
✓ Sin frameworks pesados (net/http estándar)
✓ Chi router para ruteo
✓ pgx/v5 para acceso directo a PostgreSQL
✓ Validación con go-playground/validator
✓ CORS restrictivo
✓ Tipado estricto (no interface{})
✓ Manejo centralizado de errores
✓ PostgreSQL + Redis
✓ Docker Compose para orquestación

ENDPOINTS DISPONIBLES
=====================

PÚBLICO:
  POST /auth/login                    - Login de usuario
  POST /auth/refresh                  - Renovar token
  GET  /health                        - Health check

PROTEGIDO (Require JWT):
  GET  /lotes                         - Listar lotes del usuario
  POST /lotes                         - Crear nuevo lote
  GET  /lotes/{id}                    - Obtener lote específico
  GET  /lotes/{id}/lecturas           - Obtener lecturas ambientales
  GET  /lotes/{id}/alertas            - Obtener alertas
  GET  /lotes/{id}/predicciones       - Obtener predicciones
  GET  /lotes/{id}/recomendaciones    - Obtener recomendaciones
  GET  /lotes/{id}/historial          - Obtener historial
  POST /devices/link                  - Vincular ESP32
  POST /reportes                       - Solicitar reporte

SEGURIDAD
=========

- JWT: HS256 con secret desde .env
- Contraseñas: bcrypt con cost 12
- Provisioning Tokens: SHA256 hash almacenado
- RLS: PostgreSQL con app.current_user_id
- CORS: Solo origen permitido en .env
- Rate Limiting: IP-based con golang.org/x/time/rate
- Validación: Todos los inputs validados
- HTTPS: Configurable en .env

PRIMEROS PASOS
==============

1. Setup:
   bash setup.sh

2. Docker (recomendado):
   docker-compose up -d

3. Verificar:
   curl http://localhost:8080/health

4. Login:
   curl -X POST http://localhost:8080/auth/login \
     -H "Content-Type: application/json" \
     -d '{"email":"user@example.com","password":"password"}'

5. Usar token:
   curl -H "Authorization: Bearer <token>" \
     http://localhost:8080/lotes
